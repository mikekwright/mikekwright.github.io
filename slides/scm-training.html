SCM

Source control is important in an organization to allow multiple users to work on the same codebase.

Within our org we are use 2 different SCM systems.

  • subversion
  • git

Subversion

To use subversion in our org you need to have it installed and then setup to support the mime-types supplied.


Our subversion system can be found at

https://vcs.fsglobal.net/svn


To use this repo you need to have the proper configuration

An example of this configuration can be found here


Note that the org is in the process of moving completely to git so we will not be discussing much in the concepts around subversion.

Git in FS

The decision to use git is based on its amazing ability to be a distributed scm, which means that every machine has a full copy of the history.
