Docker


Q. What is docker?


A. A service that allows you to deploy an application with all of it's dependencies as a single unit.


What is docker...

So Why Docker


Within our org we already use tools that manage dependencies, both with node (using npm) and with java (using maven). Why do we need docker?


Docker solves the problem of dependencies that you have on the OS level, rather than at the application level. These include some annoying issues.

  • Packages

    The package you want installed doesn't exist anymore

  • Security

    The OS version you use is outdated but can't wait for ops upgrade

  • Resource Allocation

    You want to share resources, but have conflicting installations

Docker Pieces


Docker is made up of two main pieces that we need to be aware of to use the service properly

  • The daemon

    This piece is responsible for fetching the different layers of an image and running the actual container

  • The client

    This is just a thin client allowing you to communicate with any daemon

Docker Architecture


Docker has 3 main items that are part of the architecture

  • Images

    Images are the initial templates for all running "containers". They are built on top of "layers" of filesystem changes

  • Registries

    Registries are the hosts of the images and the layers that make up the images providing an alias based solution to pull images

  • Containers

    Built from images and contains information around how to run, what configuration to apply. This adds a read-write layer on top of the image.

Docker Installation


Docker currently only has one available daemon, and that daemon runs on linux machines. So for linux installations the process can be simple.


Windows and Mac users instead can use the native client, but must run the the daemon within a linux environment. This can be accomplished by using a virtual machine (or pre-built tool)

  • Boot2Docker
  • Kitematic (Mac Only)
